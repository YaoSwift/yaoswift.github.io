使用MAC搭建Github+Hexo博客教程
记录一些搭建时遇到的小坑
发布日期:   2019-06-11
文章字数:   1.3k
阅读时长:   5 分
阅读次数:  

1 本地环境的安装

1.1 安装Node.js

直接使用客户端安装即可，进入下载页。根据提示安装，安装完成后打开终端进行测试，出现下述信息即表示安装成功（版本不一样出现的数字也不一样）。

node -v
v10.16.0
npm -v
6.9.0

1.2 安装Git

进入下载页。跟上一步骤同样，安装成功后，打开终端进行测试。

Git --version
git version 2.21.0

1.3 安装Hexo

注意!一定要加上sudo。后续如有的命令执行报错，加上sudo再试一次即可。这可能是由于直接使用客户端安装的NOde.js，没有获取到全局权限，每次执行需要手动加上。

sudo npm install -g hexo

2 创建本地博客资源

2.1 创建博客资源

创建一个文件夹，名字随意英文即可。打开终端，使用cd命令，进入到创建的文件夹中。如果文件夹上一级目录是‘公共’或‘下载’等系统文件夹，翻译成英文即可。如，在公共文件夹下创建profile文件夹后，打开终端输入：

cd Public
cd profile

初始化博客

hexo init

安装npm

sudo npm install

2.2 生成网页资源并开启本地服务器

hexo g //生成静态文件
hexo s //启动服务器。默认情况下，访问网址为：http://localhost:4000/

能正常访问网页即标志到此为止的步骤都成功。

3 同步到Github

3.1 同步文件

登陆Github，创建仓库，名称必须是user.github.io，如yaoswift.github.io

打开之前创建的profile文件夹，打开里面的_config.yml文件，使用文本编辑打开（也可以直接下载一个编辑器，会有代码高亮显示，如Brackets，Adobe推出的免费/开源编辑器）。修改文件最后的deploy。

注意type、repository、branch后均有空格。
打开终端，定位到profile文件夹输入以下命令。

hexo g //生成静态文件
hexo d //部署网站

执行hexo g出错则执行npm install hexo --save
若执行hexo d出错则执行npm install hexo-deployer-git --save
执行hexo d时，会提示输入Github的账号密码，跟着提示输入即可。

3.2 绑定ssh key

在Github上添加ssh key后就不用每次使用hexo都要输入账号密码了。
生成ssh key

ssh-keygen -t rsa -C "your_email@exampl"

打开终端输入open ~/.ssh，会弹出来一个窗口，将里面的id_rsa.pub文件使用文本编辑模式打开，复制里面的全部内容。
登陆Github,打开setting，进入SSH。

点击NEW SSH key，Title里填写随意，将复制的内容粘贴到key中，点击Add key完成添加。
至此基本的部署已经完成，接下来就是写博客和优化的事情了！

4 更换博客主题

对于等靠要的小白来说，原始主题这么丑，又不会自己写，那当然是直接拿别人的啦~这里本小白用的是hexo-theme-matery的主题，这个主题自身集成了很多插件，如评论、搜索、代码高亮等，很是方便，界面也挺好看的。需要其他的直接google一下hexo主题即可。

5几个小坑

5.1

第一个就是权限的问题了，在网上搜了半天没什么容易的根治方法，只要记得加上sudo就好，也不麻烦，特别是在下载安装一些插件的时候。大概就是出现以下错误的时候

Unhandled rejection Error: EACCES: permission denied, open '/Users/marknorgate/.npm/_cacache/tmp/a449402d'

5.2

第二个问题我也没有找到什么解决方法,出现以下警告

node-pre-gyp WARN Pre-built binaries not installable for fsevents@1.2.9 and node@10.16.0 (node-v64 ABI,

可以尝试以下命令

sudo npm install -g --unsafe-perm node-sass

5.3

第三个其实不算是问题，当时还查了半天发现其实一点妨碍也没有。使用主题时，安装一些插件如代码高亮插件时也会报权限错误，即使加上sudo也没用。但其实只要加上sudo就行了，不用管报错，它其实已经安装好了。

5.4

安装代码高亮插件后，弄了半天也高亮不起来！
最后发现写代码块是要指定语言的！
代码块格式：

\```[语言名称]
代码内容
\```

如

\```html
<!DOCTYPE HTML>
<html lang="zh-CN">
<head><meta name="generator" content="Hexo 3.8.0">
    <meta charset="utf-8">
    <meta name="keywords" content="使用MAc搭建Github+Hexo教程, SwiftYao">

\```

显示即为

<!DOCTYPE HTML>
<html lang="zh-CN">


<head><meta name="generator" content="Hexo 3.8.0">
    <meta charset="utf-8">
    <meta name="keywords" content="使用MAc搭建Github+Hexo教程, SwiftYao">

附：Hexo Markdown 简明语法手册

tip:作为小白，并没有发现npm等命令是应该写啥语言所以可以看到上面的都没有高亮。这个问题希望以后可以解决。
tip2:更新！作为小白，查看了代码高亮支持的语言，现在选用的是Shell session，有点偏差但是还可以，起码比啥也不亮好多了233333.
tip3:再更新！为啥我在编辑器里面看到代码高亮了，部署到服务器上就没有了？？？弱小可怜又无助T_T
